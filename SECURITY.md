# Security Policy

## Supported Versions

Security fixes are applied to the latest release and the current `main` branch.
Older releases may not receive backports.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability.

Email the maintainer at `im@ndzn.me` with:

- A description of the vulnerability and its impact
- Steps or a minimal example that reproduces it
- Affected browser, operating system, and commit or release
- Any suggested mitigation, if available

You should receive an acknowledgement within seven days. Please allow time to
investigate and prepare a fix before publishing details.

Do not include real user media, credentials, tokens, or other sensitive data in
the report.
